# Specification

## Summary
**Goal:** Clarify across the app that sign-in uses Internet Identity only, and that Google login is not supported.

**Planned changes:**
- Update the navbar unauthenticated sign-in button label to explicitly reference Internet Identity.
- Update the RequireAuth gated-access sign-in card copy to state users will sign in with Internet Identity and remove any implication that Google login is available.
- Add a Help page FAQ entry explaining Google login is not supported and that Internet Identity is the supported sign-in method.

**User-visible outcome:** Users see consistent English messaging that they must sign in with Internet Identity, and can read a Help/FAQ note confirming Google login is not available.
